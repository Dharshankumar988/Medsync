# Contributing to MedSync

First off, thank you for considering contributing to MedSync! 

## 1. Branching Strategy
- `main` is the production-ready branch.
- `dev` is the active development branch.
- Prefix your feature branches: `feat/add-ai-chat`, `fix/login-bug`, `docs/update-api`.

## 2. Commit Convention
We follow conventional commits:
- `feat:` A new feature
- `fix:` A bug fix
- `docs:` Documentation only changes
- `refactor:` A code change that neither fixes a bug nor adds a feature
- `test:` Adding missing tests

## 3. Pull Requests
- Point all PRs to the `dev` branch.
- Ensure GitHub Actions (CI) passes successfully before requesting a review.
- Include a summary of what was changed and any related Issue numbers.

## 4. Coding Standards
- **Python**: Follow PEP 8. Use `black` and `isort`.
- **TypeScript**: Use strict mode. Follow standard `eslint` and `prettier` rules.
- **Solidity**: Follow standard OpenZeppelin naming conventions.

Thank you for helping improve the future of decentralized healthcare!
